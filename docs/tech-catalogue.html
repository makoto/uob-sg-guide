Frontend / Viewers

deck.gl Viewer (docs/viewer-deckgl.html)

Buildings extruded from GeoJSON. 29 toggleable layers plus walkability grid and 5 UNA gravity layers; 39 choropleth metrics.

TechnologyVersionPurpose
deck.gl 9.x GPU-accelerated WebGL2 rendering of GeoJSON layers, extruded building polygons (GeoJsonLayer), bitmap raster overlays (BitmapLayer), text labels (TextLayer), and choropleth fills. ~700 KB CDN bundle.
MapLibre GL JS 4.x Vector basemap rendering (CARTO GL styles: dark-matter, positron, voyager). Open-source fork of Mapbox GL JS. No access token required with CARTO basemaps.
CARTO GL Basemaps — Vector basemap styles for MapLibre: Dark (dark-matter-gl-style), Light (positron-gl-style), Voyager (voyager-gl-style). Free tier, OpenStreetMap-based.

Common

TechnologyVersionPurpose
HTML / CSS / JavaScript ES6+ Single-page viewer apps. No build tools, no frameworks — plain vanilla JS loaded from CDN.

Data Formats

FormatExtensionUseNotes
GeoJSON .geojson Primary vector format for all spatial layers WGS84 (EPSG:4326). FeatureCollection with typed geometry (Point, LineString, Polygon, MultiPolygon).
CSV .csv Tabular data (census, resale prices, subzone summary) UTF-8 with header row. Fields match GeoJSON properties for parity.
PNG .png Satellite raster overlays Georeferenced by bounding box in viewer code. Colour-ramped by GEE. 1024px resolution. Rendered via SingleTileImageryProvider.
Shapefile .shp/.dbf/.shx/.prj Temporary format for ZenSVI boundary input Not committed; generated at runtime by GeoPandas.

CI/CD & Deployment

ServiceConfigurationPurpose
GitHub Actions .github/workflows/deploy.yml Build pipeline: installs Python deps, runs 3 scripts (src/extract_district.py, src/generate_subzone_summary.py, src/filter_district_layers.py) for all 5 districts, uploads docs/ as Pages artifact.
GitHub Pages Automatic from Actions Hosts the static site (viewer, index, catalogues, all GeoJSON/PNG/3DTiles).

src/fetch_global_layers.py is NOT in CI — it requires the zensvi conda env and GEE authentication. Run manually; outputs are committed to docs/geo/global/.
