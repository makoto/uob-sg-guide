Gaps & Limitations

No noise pollution data — would require primary data collection
No pedestrian foot traffic counts — could estimate via Mapillary or field observation
No sentiment/perception data — requires primary research (surveys, interviews)
Microclimate data is coarse — weather stations don't capture Queenstown-specific microclimates
DMP25 not yet machine-readable — must rely on MP2019 GeoJSON + DMP25 PDF/web
Some CSV datasets need geocoding — HDB Property Info, Schools (use OneMap Search API)

Subzone Summary

Generated by generate_subzone_summary.py. Spatially joins the above datasets plus population census and resale prices to the 15 Queenstown subzones.

Output FileFormatKey Fields
queenstown-subzone-summary.geojsonGeoJSONpopulation_density, amenity_density, amenity_count, mrt_station_count, cycling_path_km, park_connector_km, resale_median_price, resale_transaction_count, building_count, hdb_count, mean_height_m, max_height_m, population_elderly_65plus, avg_hdb_year, total_dwelling_units, dwelling_density, lst_mean_c, ndvi_mean, ndbi_mean, ghsl_height_mean, dsm_mean, dem_mean, canopy_cover_pct, canopy_loss_pct, gvi_mean, intersection_density, transit_access_score, destination_accessibility, walkability_index
queenstown-subzone-summary.csvCSV

Used by the 3D Viewer choropleth heatmap: select a metric from the dropdown to colour all 15 subzones on a YlOrRd ramp. 25 metrics available: population density, elderly share (65+, derived), amenity density, MRT stations, cycling paths, park connectors, resale flat price, avg building height, HDB blocks, total buildings, max building height, resale transactions, avg HDB year built, dwelling units, dwelling density, plus 6 remote sensing metrics — Vegetation (NDVI), Built-up (NDBI), Land Surface Temp (°C), Tree canopy cover (%), GHSL building height (m), Surface elevation (m), plus 4 walkability metrics — Intersection density, Transit access, Destination access, Walkability (BEH-NWI composite z-score). Derived fields: avg_hdb_year (mean HDB year completed), total_dwelling_units (sum of HDB dwelling units), dwelling_density (dwelling units per km²). Remote sensing fields merged from queenstown-remote-sensing.geojson: lst_mean_c, ndvi_mean, ndbi_mean, ghsl_height_mean, dsm_mean, dem_mean, canopy_cover_pct, canopy_loss_pct, gvi_mean. Walkability fields merged from queenstown-walkability.json: intersection_density, transit_access_score, destination_accessibility, walkability_index (BEH-NWI z-score composite).

Global Layers (OSMnx / GEE / Mapillary) (global/)

Generated by fetch_global_layers.py and generate_walkability.py from global data sources. Requires zensvi conda environment (osmnx, networkx, shapely). fetch_global_layers.py additionally requires GEE authentication and a Mapillary API key. Both scripts are run manually (not in CI). generate_walkability.py produces a subzone-level JSON and a 100 m walkability grid GeoJSON (2,178 cells).
