Gaps & Limitations

No noise pollution data — would require primary data collection
No pedestrian foot traffic counts — could estimate via field observation
No sentiment/perception data — requires primary research (surveys, interviews)
Microclimate data is coarse — weather stations don't capture Queenstown-specific microclimates
DMP25 not yet machine-readable — must rely on MP2019 GeoJSON + DMP25 PDF/web
Some CSV datasets need geocoding — HDB Property Info, Schools (use OneMap Search API)

Processed District Layers

Output files use {district}- prefix and are organised by data provenance: government data in docs/geo/gov-sg/, global/OSM data in docs/geo/global/, boundary/summary files in docs/geo/, and 3D tilesets in docs/3dtiles/{district}/. Spatially joins the above datasets plus population census and resale prices to each district's subzones (Queenstown: 15, Bishan: 3, Outram: 4, Tampines: 5, Newton: 6). Global Layers (OSMnx / GEE) (global/)

Generated by fetch_global_layers.py --district {district} [tasks...] and generate_walkability.py --district {district} from global data sources. Both scripts derive the bounding box from the district boundary GeoJSON. fetch_global_layers.py has 3 default tasks: buildings (OSM footprints + HDB enrichment via osmnx), osmnx (street network), gee (remote sensing). Two additional tasks: raster_pngs (GEE raster PNG overlays) and rs_grid (100m RS grid cells). Requires zensvi conda environment (osmnx, networkx, shapely). The gee, raster_pngs, and rs_grid tasks additionally require GEE authentication. Both scripts are run manually (not in CI). Buildings, street network, remote sensing (subzone-level + RS grid + raster PNGs), and walkability now exist for all 5 districts.
