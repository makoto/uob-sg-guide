Remote Sensing & Terrain

TermDefinition
DEM Digital Elevation Model. A raster grid of bare-earth surface elevations (ground level, with buildings and trees removed). This project uses SRTM DEM at 30 m resolution. Values are in metres above sea level. Queenstown range: ~4–73 m.
DSM Digital Surface Model. Like DEM but includes the tops of buildings, trees, and other objects above ground. The difference (DSM − DEM) approximates building/canopy height.
SRTM Shuttle Radar Topography Mission. A NASA mission (2000) that produced near-global elevation data at 30 m resolution. Used in this project via Google Earth Engine (USGS/SRTMGL1_003) for DEM and slope computation.
NDVI Normalised Difference Vegetation Index. Measures vegetation greenness from satellite imagery. Calculated as (NIR − Red) / (NIR + Red). Values range from −1 to +1; higher values indicate denser vegetation. Derived from Sentinel-2 imagery via GEE.
NDBI Normalised Difference Built-up Index. Measures built-up (impervious) surface intensity. Calculated as (SWIR − NIR) / (SWIR + NIR). Higher values indicate more built-up area; lower/negative values indicate vegetation or water.
LST Land Surface Temperature. The radiative temperature of the ground surface as measured by thermal satellite sensors. This project uses Landsat 8/9 thermal band data processed via GEE, reported in °C. Useful for identifying urban heat islands.
Canopy Cover Percentage of land area covered by tree canopy. Derived from the Global Forest Canopy Height dataset or similar GEE sources. Expressed as 0–100%.
GHSL Global Human Settlement Layer. An EU Joint Research Centre dataset providing global building height, built-up area, and population estimates from satellite imagery. Used in this project for subzone-level average building height.
GEE Google Earth Engine. A cloud platform for planetary-scale geospatial analysis. Used in this project to compute NDVI, NDBI, LST, canopy cover, DEM, slope, and GHSL metrics from satellite imagery.
Slope Terrain steepness computed from the DEM, expressed in degrees (0° = flat, 90° = vertical). Calculated using ee.Terrain.slope() in GEE. Queenstown range: 0–39°. Used in the slope-penalised walkability analysis.
RS Grid Remote Sensing Grid. A 100 m × 100 m grid covering each district, with per-cell values for NDVI, NDBI, LST, canopy %, DEM, and slope. Viewable in the map viewer with 5 colour modes.
NIR / SWIR Near-Infrared / Short-Wave Infrared. Spectral bands captured by satellite sensors. NIR reflects strongly off vegetation; SWIR reflects off built-up surfaces. Used to compute NDVI and NDBI.

Walkability & Network Analysis

TermDefinition
BEH-NWI Built Environment & Health – Neighbourhood Walkability Index. A composite walkability score based on z-score standardisation of 4 components: population density, intersection density, transit access, and destination accessibility. Used by this project to rank subzones. Methodology adapted from Frank et al. (2010).
Intersection Density Number of street intersections (walk-network nodes with degree ≥ 3) per unit area. Higher density indicates a more connected, walkable street grid. Computed via spatial kernel method: counts qualifying nodes within a 300 m radius, divided by circle area.
Transit Access Proximity to public transit stops (MRT exits). Measured as the network distance from each grid cell to the nearest MRT exit using Dijkstra’s shortest path algorithm on the walk network. Lower distance = better access.
Destination Accessibility Proximity to daily-needs destinations (hawker centres, supermarkets, parks, clinics). Computed as average network distance to the nearest destination across multiple categories.
Tobler’s Hiking Function An empirical model that predicts walking speed based on terrain slope. Used to penalise steep gradients in walkability analysis.
speed (km/h) = 6 × exp(−3.5 × |slope + 0.05|)
Flat ground ≈ 5 km/h; 10% uphill ≈ 3.6 km/h; optimal 5% downhill ≈ 6 km/h. Slope = rise/run (positive = uphill).
Slope-Adjusted Walkability Walkability metrics recalculated using Tobler’s hiking function. Instead of flat network distance, edges are weighted by estimated walking time based on gradient. Hilly areas (e.g., Kent Ridge) score lower; flat areas remain similar.
Betweenness Centrality A network metric measuring how often a road segment lies on the shortest path between all pairs of nodes. High-centrality edges are key corridors that channel traffic flow. Computed via OSMnx on the drive network.
Dijkstra’s Algorithm A graph algorithm for finding shortest (or least-cost) paths from a set of source nodes to all reachable nodes. Used in walkability analysis with edge weights of either distance (flat) or Tobler travel time (slope-adjusted).
Walk Network A pedestrian network graph from OpenStreetMap, including footpaths, park paths, and pedestrian bridges — distinct from the drive network used for street centrality. Downloaded via OSMnx.
UNA Urban Network Analysis. Gravity-based accessibility model from MIT City Form Lab (madina library). Measures how accessible each building is to nearby destinations (food, transit, green, health, education) weighted by distance decay. Parameters: search_radius = 1500 m, beta = 0.003.
Gravity Accessibility An accessibility measure where destinations are weighted inversely by distance (exponential decay). Closer destinations contribute more. Computed per building via the UNA/madina framework.

Geospatial & Data Formats

TermDefinition
GeoJSON An open standard format for encoding geographic data structures (points, lines, polygons) using JSON. The primary data format used in this project for all vector layers.
OSM / OSMnx OpenStreetMap — a collaborative open map database. OSMnx is a Python library for downloading and analysing street networks and building footprints from OSM. Used for buildings, walk/drive networks, and intersection density.
ECEF Earth-Centred, Earth-Fixed. A 3D Cartesian coordinate system with origin at the Earth’s centre.
WGS84 World Geodetic System 1984. The standard coordinate reference system for GPS and web mapping (latitude/longitude). All GeoJSON files in this project use WGS84 (EPSG:4326).
EPSG:4326 / EPSG:4978 Coordinate reference system codes. EPSG:4326 = WGS84 geographic (lon/lat). EPSG:4978 = WGS84 geocentric (ECEF xyz).
POI Point of Interest. A specific location of interest such as a hawker centre, MRT exit, park, clinic, or school. POIs are the destination inputs for walkability and UNA accessibility analyses.
GVI Green View Index. A measure of visible greenery from street-level imagery (e.g., Mapillary). Calculated as the proportion of green pixels in a panoramic image. Higher GVI indicates more perceived greenery.

Singapore-Specific Terms

TermDefinition
HDB Housing & Development Board. Singapore’s public housing authority. HDB flats house ~80% of the resident population. In this project, HDB property data (year completed, dwelling units, block number) is joined to OSM building footprints.
URA Urban Redevelopment Authority. Singapore’s national land-use planning agency. Provides the Master Plan subzone boundaries and height control data used in this project.
MRT Mass Rapid Transit. Singapore’s metro rail system. MRT exit locations are used as transit access points in walkability analysis.
CHAS Community Health Assist Scheme. A government subsidy programme for medical/dental care at participating clinics. CHAS clinic locations are used as health-access destinations in UNA analysis.
NParks National Parks Board. Manages Singapore’s parks, nature reserves, and greenery. NParks data includes park boundaries, nature reserves, park connectors, walking tracks, and park facilities.
ABC Waters Active, Beautiful, Clean Waters. A PUB (Public Utilities Board) programme that transforms waterways and reservoirs into vibrant community spaces. ABC Waters polygons show programme areas in each district.
LTA Land Transport Authority. Manages Singapore’s land transport infrastructure. Provides cycling path and public transport data via LTA DataMall.
Subzone The smallest administrative unit in Singapore’s planning hierarchy (Region > Planning Area > Subzone). Defined by URA in the Master Plan. Queenstown has 15 subzones (e.g., Tanglin Halt, Ghim Moh, Margaret Drive).
Planning Area / District Mid-level administrative unit in Singapore’s planning hierarchy. This project studies 5 planning areas: Queenstown, Bishan, Outram, Bukit Merah, Newton.
Hawker Centre A government-managed open-air food court offering affordable cooked food. A key component of Singapore’s food accessibility and social infrastructure.

Statistics & Visualisation

TermDefinition
Z-Score A statistical measure indicating how many standard deviations a value is from the mean. Used in the BEH-NWI walkability index to standardise components before summing. Formula: z = (x − mean) / std.
Choropleth A thematic map where geographic areas (subzones) are shaded in proportion to a data variable. The viewer uses a 5-step YlOrRd (yellow-orange-red) colour ramp for 33 choropleth metrics.
Colour Ramp A gradient of colours used to represent data values on a map. The RS grid uses 6-step ramps (e.g., green for NDVI, blue-to-red for DEM, yellow-to-green for slope). The choropleth uses a 5-step YlOrRd ramp.
CNT / RATE / IDX / SAT Metric type badges used in the choropleth panel. CNT = count (raw totals). RATE = rate or density (normalised by area or population). IDX = index or score (composite/derived). SAT = satellite-derived (remote sensing).
Feature Count The number of geographic features (points, lines, or polygons) in a dataset. Shown in parentheses next to each layer name in the viewer, e.g., “Hawker centres (9)”.
